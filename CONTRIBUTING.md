# Contributing to OpenCode Game Studio - Godot 2D

Thank you for your interest in contributing!

## Ways to Contribute

### Report Bugs
Open an issue with:
- Clear title
- Steps to reproduce
- Expected vs actual behavior
- Environment (Godot version, OS)

### Suggest Features
Open a discussion or issue:
- Describe the feature
- Why it's useful
- How it should work

### Submit Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes
4. Test locally
5. Commit with clear messages
6. Push and create a PR

## Coding Standards

### Agent Definitions
- Use YAML frontmatter for metadata
- Keep descriptions concise (< 200 chars)
- Document delegation map

### Documentation
- Use clear headers
- Include code examples
- Reference official docs

### Godot Code
- Follow GDScript standards in `docs/godot-2d-patterns.md`
- Use static typing
- Include comments for complex logic

## Commit Messages

```
type(scope): short description

- Detail 1
- Detail 2
```

Types: `feat`, `fix`, `docs`, `refactor`, `style`

## Questions?

Open a discussion for questions about contributing.
